# 文字游戏框架交接文档

## 今天完成了什么

- 在干净 Godot 4.7 项目里搭了一个轻量玩法框架。
- 玩家操纵目标文字 `我`，按 60px 网格移动。
- 所有文字实体都有格子碰撞，玩家不会和字重叠。
- 实现了屏幕页切换：逻辑地图固定，玩家跨出 32x18 当前页后，显示层一次性平移到下一页。
- 实现并测试了空格互动、退格删字、推字、Alt 拉字、Tab 拆字、组字、造句识别和高亮。
- 明确了 Alt 拉字规则：按住 Alt 时只能朝远离被拉文字的方向移动。比如 `我` 在字上方，只能 Alt + 上；Alt + 左、右、下都不允许，除非松开 Alt。
- 明确了自动合字规则：不是调用测试函数手动合成，而是玩家推着一个偏旁往另一个偏旁上挤时自动合成。例如推 `又` 撞 `戈`，生成 `戏`。
- 明确了造句提示规则：`已识别` 这类提示会生成成地图上的文字实体，有碰撞，后续也可以给其中某些字补交互。
- 明确了普通提示规则：例如 `手表可以查看人类世界的时间` 也会作为地图文字实体出现，不自动消失，默认有碰撞。重复触发同一句提示时复用已有实体，不刷出一堆重复提示。
- 增加了 F5 自动演示路线，方便组里快速看机制。
- 增加了 headless 自动测试，方便后续重构时回归。
- 增加了 `tools/run_all_tests.ps1` 一键验证脚本，包含自动打开 Godot、截图、保存画面和基础像素检查。
- 画面字体已切换为 `Fonts/Zpix.ttf`，视觉更接近原版。
- 新增 `docs/测试复用指南.md`，后续按截图继续做关卡时优先看这份。
- 头盔关里的 `树` 字现在复用参考源码 `Scenes/Animations/TreeSprite.tscn` 的 20 帧循环动画；`木` 仍保持静态字形。

## 关卡数据格式

当前测试关卡在 `scripts/level_loader.gd`。

`rows` 是文本网格：

- 一个中文字符默认占一个格子。
- 空格表示空地。
- `我` 表示说明上的玩家起点，但真实起点以 `player_start` 为准。
- 多字词例如 `手表` 会作为一个实体解析，占连续两个格子。

`entities` 用来覆盖默认属性：

```gdscript
"石": {"pushable": true, "solid": true}
"删": {"deletable": true, "solid": true}
"手表": {"interact_text": "手表可以查看人类世界的时间", "solid": true}
```

规则配置：

```gdscript
"split_rules": {"戏": ["又", "戈"]}
"merge_rules": {"又+戈": "戏", "戈+又": "戏"}
"sentence_rules": {"天气": {"message": "已识别"}}
```

当 `sentence_rules` 识别成功时，`message` 不只是 UI 字幕，还会生成地图文字实体。默认生成在被识别词附近的空格上，且 `solid=true`。后续可以扩展：

```gdscript
"sentence_rules": {
	"天气": {
		"message": "已识别",
		"caption_pos": Vector2i(16, 9),
		"caption_interact_text": "这里可以添加特殊交互"
	}
}
```

普通交互提示同理：`interact_text` 会调用同一套地图提示生成逻辑，所以可以直接用于测试提示文字碰撞。后续如果需要让提示文字可删、可推或可互动，可以继续扩展提示配置。

后续让 Codex 做关卡时，推荐输入：

1. 原游戏截图。
2. 这张图对应的文字地图草稿。
3. 哪些字可推、可删、可拆、可互动。
4. 本关拆字、合字、造句规则。
5. 目标演示路线。

## 运行和测试

打开 Godot 4.7，运行 `Main.tscn`。

控制：

- 方向键：移动。
- Space：互动。
- Backspace：删字。
- Tab：拆字。
- Alt + 方向键：拉字。
- F5：自动演示。

命令行测试：

```powershell
.\tools\run_all_tests.ps1
E:\Godot\Godot_v4.7-stable_win64_console.exe --headless --path "E:\wordgame copy\新建游戏项目" -s res://tests/test_gameplay_core.gd
E:\Godot\Godot_v4.7-stable_win64_console.exe --headless --path "E:\wordgame copy\新建游戏项目" -s res://tests/test_precision_movement.gd
```

## 后续建议

- 先保留 `GridWorld` 作为唯一玩法真相来源，输入、演示和测试都调用它。
- 地图规模变大后，把 `LevelLoader.build_test_level()` 拆成多个关卡资源或 JSON 文件。
- 如果要更像原作，再单独做文字渲染层；不要把旧项目的大量全局单例直接搬进来。
- 如果中文显示异常，优先加入可分发的开源 CJK 字体，不复制原游戏字体或素材。
- 每做一个新机制，先在 `tests/test_gameplay_core.gd` 增加回归测试。
