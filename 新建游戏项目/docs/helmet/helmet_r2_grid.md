## 第一关复刻方法总结

第一关确定下来的复刻流程可以继续复用：

1. 先把截图切成 32 x 18 的逻辑格子，所有静态文字、水、树都写入 `rows`。
2. 玩家 `我` 不直接作为普通实体写入逻辑层；如果截图中 `我` 是玩家，就用 `player_start` 指向该格，避免出现两个 `我`。
3. 关卡中的特殊可推字用 `cell_entity_configs` 按坐标覆盖，避免同字不同位置互相污染。
4. 合字/拆字走核心 `merge_rules` / `split_rules`，核心会记住合成来源格，拆开时回到原位。
5. 关卡视觉联动用 `merge_effects` / `split_effects`：移除旧格、生成桥、恢复溪和树、替换提示文字。
6. 长句如果只是提示，用非阻挡文字；如果句中某些字要参与解谜，就用 `spawn_text` 的 `as_chars: true` 按格逐字生成。
7. 每关新增专项测试，锁住初始尺寸、关键坐标、合成状态和拆回状态。

第一关地图仍保存在 `res://levels/helmet/helmet_r1.gd`，没有被第二关覆盖。

坐标系：左上角为 `(0,0)`，`x` 向右，`y` 向下。地图尺寸为 32 列 x 18 行。

玩家初始位置：`(6,4)`，位于第二行 `感觉` 前面的空格。


## 解谜流程

第一步：将句子中的 `乔木` 合成 `桥`。

```text
乔：(14,3)
木：(15,3)
```

合成后：

- 句子中的 `乔木` 折叠为 `桥`。
- 右侧树形消失。
- 河面生成一段偏左的桥，但这段桥仍然过不了溪。
- 左侧距离提示句此时仍然不出现。

偏左桥：

```text
(16,7), (24,7)
(17,8) 到 (23,8)
(17,10) 到 (23,10)
(16,11), (24,11)
```

中间道路：

```text
(18,9) 到 (23,9)
```

但 `(24,9)` 和 `(25,9)` 仍然是 `溪`，所以无法直接通过。

第二步：面向这段过不了河的 `桥` 按 Space，左侧生成距离提示句，其中 `一` 和 `二` 是可推动的解谜字。

距离提示句：

```text
这桥 肯定帮得上忙。
再往溪流靠近一大步，
离对岸就剩约二十尺。

搞定距离即解决问题。
```

`这桥 肯定帮得上忙。` 中 `桥` 和 `肯` 之间保留一个空格，用来给拆桥时的 `木` 回到原位。

第三步：把 `二` 推向 `一`，合成 `三`。

变化前：

```text
再往溪流靠近一大步，
离对岸就剩约二十尺。
```

变化后：

```text
再往溪流靠近三大步，
离对岸就剩约 十尺。
```

合成 `三` 后，`离对岸就剩约十尺。` 这一行不整体移动，只把原来 `二` 的位置留空，因此 `约` 和 `十` 中间有一格空白。桥移动到第一关的可通行桥位置。

可通行桥：

```text
(19,7), (27,7)
(20,8) 到 (26,8)
(20,10) 到 (26,10)
(19,11), (27,11)
```

中间道路：

```text
(21,9) 到 (25,9)
```

## 交互提示

```text
无法跨越的湍急野溪，
必须更努力造出路来。
```

```text

小心点肯定帮得上忙。
```

面向偏左、过不了河的 `桥` 按 Space 后，生成距离提示段。该提示段出现后不会自动消失。
