## 交互提示

面向任意 `溪` 按 Space 后，在左侧 `(2,9)` 起生成两行：

```text
无法跨越的湍急野溪，
必须想办法造一条路。
```

面向树形中的 `树` 或 `木` 按 Space 后，在左侧 `(2,6)` 起生成两行：

```text
这乔木看起来很结实，
肯定能够帮得上忙吧。
```

这些提示字目前配置为不阻挡移动，只用于显示说明。

## 乔木合桥

第二行开场文字中的玩家 `我` 是起点；第一行末尾的 `乔木` 位于：

```text
乔：(14,3)
木：(15,3)
```

这两个字可以被推动并合成为 `桥`。合成成功后：

- 文字句子里的 `乔木` 会折叠为 `桥`。
- 右侧树形会消失。
- 河面生成上下两排阻挡移动的 `桥` 字作为桥边，中间清出一行空格作为可通行道路。

河面桥字生成格：

```text
(19,8), (27,8)
(20,9) 到 (26,9)
(20,11) 到 (26,11)
(19,11), (27,11)
```

其中 `(21,9)` 到 `(25,9)`、`(21,11)` 到 `(25,11)` 原本是 `溪`，合桥时会被桥替换。

中间道路格：

```text
(21,10) 到 (25,10)
```

这些格子原本是 `溪`，合桥时会清空为可通行道路，不生成 `桥` 字。

如果树提示已经被交互生成，第一行提示中的 `乔木` 也会同步变化：

```text
这乔木看起来很结实，
```

合桥后变为：

```text
这桥看起来很结实，
```

拆回后再恢复为 `这乔木看起来很结实，`。

拆 `桥` 时会回到原来的 `乔木` 两格，而不是按玩家朝向临时生成：

```text
乔 回到 (14,3)
木 回到 (15,3)
```

如果玩家站在 `(15,3)`，也就是 `桥` 右侧、`木` 应该回来的位置，拆字时会先把玩家向右推到 `(16,3)`，再把 `木` 放回 `(15,3)`。

拆回 `乔木` 后，河面桥消失，右侧树形恢复，溪水也恢复到原来的格子。
