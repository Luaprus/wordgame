# 《文字游戏》三大关卡复刻需求整理版

来源：`E:\wordgame copy\docs\文字游戏三大关卡像素级复刻需求文档.md`

本文是原始需求的工程化整理版，用于驱动 `harness/features.json`、测试矩阵、视觉验收和最终验收。若本整理版与原始需求冲突，以原始需求为准；若实现中发现需求缺口，必须先补充基准资料或人工标注，不允许推测验收。

## 目标

在 Godot 4.7 中复刻 Steam 版《文字游戏》指定章节，目标不是“可玩”或“大致相似”，而是在目标范围内达到：

- 像素级一致：布局、字体、字号、颜色、镜头、截图状态与原版一致。
- 行为级一致：输入、移动、碰撞、交互、失败、状态切换、敌人和机关行为与原版一致。
- 音画级一致：动画帧、过渡、音效、BGM、闪白、震动、打字机节奏与原版一致。

任何“基本复刻”“接近”“大致一致”都不能作为验收口径。

## 强制验收原则

1. 必须先建立原版基准库，再开始关卡复刻。
2. 每个基准项必须能追溯到录屏、截图、源码或人工标注。
3. 每个复刻实现必须映射到一个或多个基准项。
4. 自动测试、自动演示、截图对比和人工复核都通过后，功能才允许完成。
5. 未经甲方批准，不允许保留“已知差异”并标记完成。
6. 没有 feature_id，不允许写代码。
7. 没有 acceptance_criteria，不允许开始实现。
8. 没有测试脚本，不允许标记完成。
9. 测试失败，不允许交付。
10. 修改超出 allowed_files，必须先更新 `harness/plan.md` 并说明原因。
11. 任何架构级改动必须先更新 `harness/contracts.md`。
12. 多人合作时，以 `harness/progress.jsonl` 为唯一进度来源。
13. 每次提交前必须从 GitHub 远端全量拉取并完成同步；未同步、未解决冲突、未重新测试，不允许提交或推送。

## 基准库交付

阶段一必须交付以下基准表：

- 视频基准表：视频元数据、目标段落、关键事件起止时间和帧号、对应关卡和源码场景。
- 截图基准表：原版截图、来源页码或时间码、状态名称、玩家坐标、镜头位置、文字布局、动态对象位置。
- 地图格子基准表：地图尺寸、格子尺寸、玩家初始状态、对象坐标、碰撞、推拉删拆合交互属性、层级、可见性条件。
- 行为基准表：输入、前置状态、成功/失败条件、原版结果、复刻结果、冷却、输入锁、动画、音效、后续状态。
- 动画基准表：来源时间码、源码场景、起止帧、关键帧、位移、透明度、颜色、缩放、闪烁、震动、shader 参数、输入锁和音效。
- 音频基准表：音频文件、源码路径、触发关卡、触发行为、触发时间、延迟、音量、声像、循环、淡入淡出、停止条件。
- 源码索引表：目标地图、脚本、事件、动画、音效和资源路径。

## 游戏基座需求

- 玩家显示为“我”，记录格子坐标、朝向、移动状态、输入冷却、动画/事件锁定、当前能力。
- 支持方向键和 WASD；移动以格子为单位，过程连续，结束停在格子中心。
- 实现“第一下转向，第二下移动”。
- 所有交互必须要求玩家面向目标字。
- Backspace 删除、Tab 拆字、推字、Alt 拉字、合字、句子规则必须与原版行为一致。
- 打字机系统生成的字必须能参与碰撞、删除、推拉、拆字、合字和句子判定。
- 句子规则需要横向/纵向扫描、匹配、设置开关/变量并播放高亮。

## 目标关卡范围

### 贝克思贝斯之剑关卡包

必须覆盖：

- 获得剑后的教学段。
- 史莱姆段。
- 蛇妖段。
- 村中溶解显现段。
- 15 个 P0 动画：删除字、句子高亮、打字机、“我”字呼吸、“爱”字替代、蛇妖摆动、受伤闪烁、房子撞碎、“仓”调到蛇上、蛇跳动、闪白、发射媚眼、蛇剥皮、蛇抖动、村中溶解显现。

### 杜尔手套关卡包

必须覆盖：

- 手套关卡全流程。
- 正确路线和错误路线。
- 填字变手势。
- 删除“不”后的放开状态。
- 手掌碰撞和通路变化。
- 每个有效手势都必须有截图、地图和碰撞基准。

### 四目头盔过河六关

必须覆盖：

- 获得头盔后的六关全流程。
- 每关初始状态、正确解法、错误/无效操作反馈。
- 六关之间连续推进和最后收尾状态。
- 拆字、组字、推字、调查、通行状态变化。
- 持续动画、合成动画、搭桥动画、拆解动画的复用判定。

## 自动化验收

自动化工具至少包含：

- 原版基准表读取。
- 复刻自动演示。
- 关键状态截图导出。
- 原图/复刻图/差异图生成。
- 基座行为测试。
- 关卡通关测试。
- 动画触发测试。
- 音频触发日志。
- 验收报告生成。

自动化验收不能替代人工复核；但没有自动化验收，不允许进入最终验收。

## 不得标记完成的情况

- 未建立原版基准库。
- 仍存在未核验项。
- 使用弱口径作为验收标准。
- 关卡可通关但画面不一致。
- 画面一致但行为不一致。
- 行为一致但动画或音效不一致。
- 玩家可从侧面或背后操作字。
- 转向第一下错误移动。
- 长按移动跳格或半格停留。
- Alt 拉字允许推方向或垂直方向移动。
- 剑关卡缺少任一 P0 动画。
- 手套关卡填字后不改变对应手势。
- 过河六关缺少任一关或任一失败反馈。
- 自动演示缺失。
- 自动测试缺失。
- 差异项未经甲方批准就被视为完成。
