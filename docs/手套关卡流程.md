# 第三章“手套关”运行全流程与讲解路径

## 文档用途

这份文档面向制作和讲解 PPT 的同学。它不是代码交接稿，而是把手套关讲成一个完整的“文字改变世界”的关卡故事：玩家先学会推、拉、删字，再通过改变句子和手势的含义，穿过巨掌迷宫、取得掌中剑，并面对勇者们的期待。

为避免把尚在制作中的内容误讲成已完成成品，本文用两种状态标记：

- **可运行**：当前根项目中已有可试玩流程。
- **已定义，待接入验证**：玩法与文案已经确定，但尚未完成正式流程验收。

当前正式试玩场景为 `E:/wordgame copy/levels/glove/glove_preview.tscn`。

## 关卡一句话讲解

手套关让玩家第一次真正理解：文字并不是说明，而是世界本身。把一个字从句子里拉出来、推到另一句话里，巨掌的姿势、道路的开闭和角色的能力都会随之改变；最后，玩家获得“删字”的力量，把“不”删掉，让巨掌张开。

## 运行全流程

### 1. 手套获得与第一次推动教学

**可运行。**

进入手套关后，先播放手套获得演出。画面恢复后，中央以打字机效果出现：

```text
不知道该如何使用这力量，又该承担怎样的责任。
我犹豫地向前踏出了一步。勇者的觉悟不言自明。
```

随后出现操作引导：

```text
「找出该被搬移的文字，用『方向键』推动看看文字吧。
展现出勇者该有的觉悟，只在一步之遥。」
```

这一段不是单纯提示。底部句子中的“我”和“不”都在格子世界里：玩家先控制“我”移动，再把“不”推入指定空位。句子会在玩家推动过程中逐字改写，说明这副手套赋予的不是普通力量，而是把文字当作实体搬动的能力。

讲解重点：

- “我”不是 UI 图标，而是一个可移动、会碰撞的字。
- 每个能出现的字都可以成为墙、道路、机关或可推动物。
- 推字时玩家留在原格，被推文字先移动一格；短暂后摇后可继续持续移动。



### 2. 拉字教学与勇者的质问

**可运行。**

第一次推动完成后，底部文字会自动改写为新的两行叙述；玩家暂时不能移动，直到文字打完。接着上方显示关于“文字陷在死角”时如何使用拉字的说明：

```text
「当文字不小心陷在死角里，用『ALT』加方向键先拉回正道再说吧！
不过这比推动费劲多了。」
```

玩家需要把“不”从句子中拉出并移动到新的位置。拉字完成后，勇者会质问：

```text
「这份觉悟够清楚，但这样子就算是勇者了吗？谁知道你是不是碰巧蒙到？
你最好再进一步发挥！」
```

这段的作用是让玩家理解“推”和“拉”是两种不同的空间操作：推适合把前方的字顶开，拉适合把卡在角落或句子内部的字带出来。

### 3. 勇者交出下一步试炼

**可运行。**

勇者随后在屏幕中央打出：

```text
勇者说：「你已经学会了使用这股力量，
现在你把剑给我，继续向前完成勇者试炼吧」
```

其中的“我”会从句子中成为可控制实体。玩家可以让“我”走出这句话，向右离开；画面闪白并震动后，进入巨掌迷宫的手势试炼。

讲解时可以把这一段解释为：玩家已经掌握了“搬字”，但还没有真正理解如何通过改写一句话来改写整个空间。

### 4. 巨掌迷宫开场：生命线是一堵可以改写的墙

**可运行。**

进入巨掌迷宫后，玩家被巨大的“掌”字墙围住。初始状态下，地图中央是一串“掌”构成的阻隔；玩家可在任意时刻确认，中央文字会从左到右逐字替换为：

```text
逼退好手的生命线
```

替换期间玩家不能移动。没有被替换到的格子仍显示“掌”，因此玩家能直接看见“文字墙”被一句话逐格改写的过程。

这句话中的“好”是可拉出的关键字。玩家把“好”从句子中拉走后，原来由“掌”构成的中间竖墙消失，出现一条可以绕行的通路；把“好”放回原句，竖墙会恢复。


| 玩家动作      | 句子/空间变化        | 讲解要点                   |
| --------- | -------------- | ---------------------- |
| 确认中央“掌”墙  | 逐字变为“逼退好手的生命线” | 句子本身就是地图机关             |
| 把“好”从句中拉出 | 中央竖向“掌”墙消失     | 抽走一个字，也会抽走它在句子里承担的空间意义 |
| 把“好”放回句中  | 竖墙恢复           | 世界会根据句子是否完整即时回退        |




### 5. 把“好”放进手势槽：从语义变化到画面变化

**可运行。**

玩家先将手势槽中的“零”拉出，再把“好”推入“是__的手势”的空格。巨掌随即切换到“好”的手势。

手势切换不是瞬间换图：画面会闪白并整体抖动约 1 秒，约在前 30% 的时间点切到新手势，随后亮度恢复。巨掌形状、可通行区域和生命线的开闭会一起改变。

此时“逼退__手的生命线”中出现空档，玩家可以从空档穿过，进入原来被封闭的区域。

讲解重点：

- 字被推入句子，等于补全新的语义。
- 语义成立后，巨掌会作出对应手势。
- 手势不是装饰，它会改变碰撞、道路和后续可取得的字。



### 6. 把“二”放进手势槽，取得掌中剑

**可运行。**

穿过“好”手势打开的空档后，玩家可以取得并搬运“二”。把“二”填入“是__的手势”的空格，巨掌变为二指手势，通向掌中“剑”的路线随之打开。

玩家靠近句中的“剑”并确认后，会获得掌中剑：

- “剑”从原格向“我”的下方移动；
- “我”遮住“剑”，表现为拿取；
- “剑”自然消失，不会跳到地图另一侧；
- 玩家获得删字能力。

“二”手势下，掌中剑还具有左右换位的机关效果：只有手势成立时，互动才会让剑在巨掌中的两个位置之间切换。

### 7. 删除“不”，让巨掌放开

**可运行。**

拿到剑后，玩家需要再把“好”放回手势槽，回到“好”的手势，穿过生命线句子中的空档，接近“不会轻易放开”里的“不”。

对“不”按 Backspace/Delete 后，画面播放删字切割动画：目标字被斜劈、闪白并消失。句子从“不会轻易放开”变成“会轻易放开”，巨掌随即张开。

这一次改写的意义比前面更强：


| 改写前    | 删除  | 改写后   | 世界结果                  |
| ------ | --- | ----- | --------------------- |
| 不会轻易放开 | 不   | 会轻易放开 | 巨掌张开，封闭区域被放开，玩家可接近“勇” |


每次巨掌手势改变，画面都会闪白、震动，再显示新的巨掌形状。这里是整关第一次明确告诉玩家：删掉一个否定字，不只是打开一格路，而是改变一个巨大的角色和空间的意志。

### 8. 与“勇”对话，进入尾声

**可运行。**

巨掌张开后，玩家终于能接近“勇”。确认互动后，原地图清屏，依次出现三页勇者对白：

```text
「果然有两把刷子，」
「你，挺不简单的。」
```

```text
「你与我们都不同，」
「也许是这样的你，」
「才能够有所突破！」
```

```text
「四三九七号勇者！」
「请无畏地上前吧！」
```

每页由空格确认进入下一页。第三页再次确认后，才会清屏并进入“勇者围拢”动画。

### 9. 勇者围拢：期待变成一条狭窄的道路

**可运行。**

围拢动画按原版场景时序迁移：三面边界先后出现大量“勇”字，分批向中央推进。动画在约 1.1 秒、2.6 秒、3.7 秒、5.5 秒进入新的阶段；每次单格推进约 0.3 秒，字与字之间保留短暂停顿。

最后，靠近中央的左右两列“勇”字渐隐，外侧空隙同步补满新的“勇”字，形成两侧和上方厚重的围墙，只在中间留下一条笔直道路。画面以打字机效果写出：

```text
勇者们整齐地排成阵，留出一条笔直的道路。
```

讲解重点：这里不是失败画面。勇者们没有把玩家挤死，而是以“期待”和“秩序”围出唯一一条向前的路，呼应整关“觉悟”的主题。

### 10. 围拢后的回大厅收束

**已定义，待接入验证。**

围拢动画完成后，计划继续以下流程：

1. 在围墙中央打字机显示“我感到温热的期许和冀望。”
2. 这句话中的“我”成为可移动实体。
3. 中下方渐显“世界由__拯救”，其中“由”后留一个“我”能进入的字格空位。
4. 当“我”走进该空位，下方渐显“公主由__解放”。
5. 当“我”再走进第二个空位，画面闪白并震动，返回大厅。

这一段的叙事作用是把前面的“勇者要求你向前”落到玩家的主动选择上：不是系统替“我”填词，而是“我”亲自进入“世界由我拯救”和“公主由我解放”的句子。



1. 不要把“推字、拉字、删字”讲成纯 UI 操作。它们都是在有碰撞的格子世界中移动实体字。
2. 不要把手势变化讲成换皮。手势会修改巨掌墙体、道路和可抵达区域。
3. 不要把“勇者围拢”讲成失败。当前画面是“被期待包围后，仍留有向前道路”的收束意象。
