# 《文字游戏》目标章节全量一致复刻需求文档 v4

## 1. 文档定位

本文档用于约束《文字游戏》目标章节在 Godot 4.7 中的复刻工作。甲方目标已明确升级为：最终成品在目标章节范围内，达到与原版 Steam 版《文字游戏》对应内容的全量一致。

“全量一致”包括：

- 像素级一致：画面布局、字体、字号、颜色、镜头、截图状态与原版一致。
- 行为级一致：输入、移动、碰撞、交互、失败、状态切换、敌人/机关行为与原版一致。
- 音画级一致：动画帧、过渡、音效、BGM、闪白、震动、打字机节奏与原版一致。

只满足玩法可通关、地图大致相似或动画效果近似，不得视为完成。

## 3. 验收原则

最终验收不能只依据需求文字。必须先建立“原版基准库”，再用复刻结果逐项对照。

验收通过条件：

1. 目标章节所有关卡均有完整基准记录。
2. 每个基准项均有原版资料来源。
3. 复刻工程对每个基准项均有对应实现。
4. 自动测试、自动演示、截图对比和人工复核均通过。
5. 所有差异均被修复；未经甲方批准，不允许保留“已知差异”作为完成项。

若某一项无法从录屏、截图或源码中确认，必须先补充资料或人工标注；不得以推测方式验收。

## 5. 游戏基座需求

游戏基座必须严格复刻原版行为。基座未验收通过，不得进入关卡验收。

### 5.1 玩家“我”

玩家对象必须显示为“我”，并记录：

- 当前格子坐标。
- 当前朝向。
- 是否正在移动。
- 是否处于输入冷却。
- 是否处于动画/事件锁定。
- 当前能力：删除、推拉、拆字、合字等。

朝向采用上、下、左、右四方向。原版源码 `BaseCharacter.gd` 中默认 `direction = 2`，方向值为下 2、左 4、右 6、上 8。复刻内部表示可以不同，但外部行为必须一致。

### 5.2 移动

必须支持方向键和 WASD。

移动要求：

- 长按方向键或 WASD 可连续移动。
- 移动以格子为单位。
- 每次移动目标为相邻一格。
- 移动过程连续，不得瞬移。
- 移动结束必须停在格子中心。
- 不得出现半格停留、漂移或累计误差。

原版源码依据：

- `BaseCharacter.gd` 使用 `now_pos` 记录目标格。
- `moving_pos` 记录移动中的连续位置。
- `update_move()` 逐帧将显示位置逼近目标格。
- 原版格子单位约为 60 像素。

### 5.3 朝向优先

必须实现“第一下转向，第二下移动”。

例：

- 当前面向下，按右键：只转向右，不移动。
- 当前面向右，再按右键：向右移动一格。

源码依据：

- `Player.gd` 中仅当输入方向等于当前方向，或满足直接改向条件，才调用 `move_straight()`；否则调用 `set_direction()` 并进入短冷却。

### 5.4 面向交互

所有交互必须要求玩家面向目标字。

适用：

- 确认交互。
- Backspace 删除。
- Tab 拆字。
- Alt 拉取。
- 推字。
- 合字。
- 句子填字。

玩家从侧面或背后不得操作目标字。

### 5.5 Backspace 删除

输入：单击 Backspace。

要求：

- 面前一格有可删除字时，执行删除或特殊删除事件。
- 删除时播放原版删除动画。
- 删除后重新检查句子和关卡状态。
- 不可删除时播放失败反馈。

源码依据：

- `Player.gd` 的 `backspace_front()`。
- `Scenes/Animations/Backspace.tscn`。
- `Scenes/Animations/Backspace_fail.tscn`。

### 5.6 Tab 拆字

输入：单击 Tab。

要求：

- 面前一格有可拆字且规则存在时，执行拆字。
- 拆出的部件作为真实地图文字存在。
- 拆字后重新检查句子和状态。
- 不可拆时播放失败反馈。

源码依据：

- `Player.gd` 的 `split_front_event()`。
- `Scenes/Animations/Unzip_fail.tscn`。

### 5.7 推字

触发：面向可推字并向该方向移动。

要求：

- 可推字沿移动方向移动。
- 不可推字阻挡玩家。
- 推动后检查句子规则。
- 连锁推动、合字、挤压规则必须按原版表现登记并复刻。

源码依据：

- `can_push`。
- `Player.gd` 推字逻辑。
- `Scenes/Animations/Push.tscn`。

### 5.8 Alt 拉字

输入：按住 Alt 面向可拉字，再输入方向。

要求：

- Alt 锁定面前可拉/可推字。
- 拉取期间只能朝“拉”的方向移动。
- 不允许朝推方向移动。
- 不允许朝垂直方向移动。
- 松开 Alt 后解除锁定。

例：

- 玩家在目标字上方，面向下并按住 Alt 锁定目标后，只能向上移动。

源码依据：

- `Player.gd` 的 `check_front_pull_event()`。
- `Player.gd` 的 `pulling_move()`。
- `Scenes/Animations/Pull.tscn`。
- `Scenes/Animations/PullLine.tscn`。

### 5.9 打字机系统

必须复刻原版打字机系统。

要求：

- 剧情/教学文字逐字出现。
- 支持清除已打文字。
- 支持等待、换行。
- 支持内联字属性。
- 生成的字可参与碰撞、删除、推拉、拆字、合字和句子判定。

源码依据：

- `Scripts\Typewriter.gdc`。
- 地图 `.tscn` 中的 `clear_typed` 和文本 commands。

### 5.10 句子规则与高亮

必须复刻句子成立规则。

要求：

- 横向/纵向扫描地图文字。
- 匹配目标句子。
- 成功时设置开关/变量。
- 成功时播放原版高亮动画。
- 字被移走或状态变化时，按原版规则撤销或保持状态。

源码依据：

- `Scripts\MainMap.gdc`。
- 地图 `.tscn` 中的 sentence rule commands。

## 7. 画面一致验收

画面一致必须以截图对比为准。

每个截图基准点必须提供：

- 原版截图。
- 复刻截图。
- 差异图。
- 差异说明。

通过标准：

- 可见文字位置一致。
- 可见对象位置一致。
- 字体、字号、颜色、行距、字距一致。
- 镜头位置一致。
- 画面比例一致。
- 无错字、漏字、多字。
- 无错误遮挡、重叠、错层。

容差：

- 默认不设置视觉容差。
- 若 Godot 3.4.2 与 Godot 4.7 渲染差异导致无法逐像素一致，必须单独列出差异类型、影响范围，并由甲方批准。

## 8. 行为一致验收

行为一致必须以行为基准表和自动测试为准。

每个行为必须验证：

- 输入。
- 前置状态。
- 成功结果。
- 失败结果。
- 冷却。
- 输入锁定。
- 状态变量。
- 后续事件。

任何未登记的行为不得算作已复刻。

## 9. 音画一致验收

音画一致必须以动画基准表和音频基准表为准。

要求：

- 主线动画必须逐帧或逐关键帧核验。
- 音效必须在正确事件点触发。
- BGM 必须在正确段落播放、停止或切换。
- 闪白、震动、溶解、淡入淡出等效果必须记录参数。

若原版素材因使用边界不能随工程分发，仍必须在本地研究版本中完成触发点和时间轴复刻；对外版本另行处理。

## 10. 自动化验收

乙方必须提供自动化验收工具。

至少包含：

- 原版基准表读取。
- 复刻自动演示。
- 关键状态截图导出。
- 原图/复刻图/差异图生成。
- 基座行为测试。
- 关卡通关测试。
- 动画触发测试。
- 音频触发日志。
- 验收报告生成。

自动化验收不替代人工复核，但没有自动化验收不得进入最终验收。

## 12. 不得视为完成的情况

出现任一情况，不得通过最终验收：

- 未建立原版基准库。
- 仍存在未核验项。
- 仍使用“基本复刻”“接近”“大致一致”等弱口径作为验收标准。
- 关卡可通关但画面不一致。
- 画面一致但行为不一致。
- 行为一致但动画或音效不一致。
- 玩家可从侧面或背后操作字。
- 转向第一下错误移动。
- 长按移动跳格或半格停留。
- Alt 拉字允许推方向或垂直方向移动。
- 剑关卡缺少任一 P0 动画。
- 手套关卡填字后不改变对应手势。
- 过河六关缺少任一关或任一失败反馈。
- 自动演示缺失。
- 自动测试缺失。
- 差异项未经甲方批准就被视为完成。
