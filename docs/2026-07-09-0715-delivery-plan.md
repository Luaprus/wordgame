## 每晚统一验收标准

每天晚上收工前必须满足：

1. 当天领取的 feature 都写入 claimed。
2. 完成项必须有 test_passed。
3. 未完成项必须写 blocked，并说明缺什么、谁接手。
4. 改动不得越过 allowed_files；如果越界，必须先更新 plan 或 contracts。
5. 至少完整执行一次 tools/run_all_tests.ps1。
6. 涉及画面、流程或交互的功能，必须提供人工验收入口：场景、demo route、debug panel 三选一。

## 人工验收入口规则

任何功能如果没有给非开发者看的验收入口，对团队来说就不算可验收。每个功能完成时必须至少提供以下之一：
- 可运行场景
- 可复现 demo route
- 调试/验收面板

## 超载止损规则

如果任一天晚上发现基座关键路径落后半天以上，第二天立刻执行止损：
- 砍非关键 polish
- 保主流程
- 保自动测试
- 保人工验收入口
- 不再新增非 P0 范围工作

## 执行原则

- 没有 feature_id，不允许写代码。
- 没有 acceptance_criteria，不允许开工。
- 没有测试脚本，不允许标记完成。
- 测试失败，不允许交付。
- progress.jsonl 是唯一进度来源。
