# 玩法流程审核页面

打开 `index.html`，再将剑关、手套关、头盔关页面分别分发给对应审核者。每个页面只有本关的数据和本关的本地草稿，不会修改其他关卡。

审核者可以随时新增情境、操作、结果和人工备注。操作的条件必须明确填写；“无条件”也要写“无”。失败、重置、终局和保持当前情境都是需要记录的结果。

状态含义：待补充表示信息不全；待复查表示已有说法但需要证据或第二人确认；已确认表示可以进入实现基线；不纳入复刻表示已确认超出范围或无关，并必须说明原因。

页面的“导出审核结果”只会下载 JSON 文件。请把该文件交给集成人员，由其使用 `tools/import_gameplay_review_result.mjs` 导入。不要手工编辑 `review_data.js` 或 `*.confirmed.json`。
