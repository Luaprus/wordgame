# 视觉验收规则

## 默认原则

- 默认逐像素一致，默认不设置视觉容差。
- 每个截图基准点必须生成原版截图、复刻截图、差异图和差异说明。
- Godot 3.4.2 与 Godot 4.7 的渲染差异不得自动豁免；必须单独登记差异类型、影响范围和甲方批准记录。

## 必检项目

| 项目 | 要求 |
| --- | --- |
| 画面比例 | 与原版截图一致 |
| 镜头位置 | 与基准截图一致 |
| 可见文字 | 内容、位置、层级、字体、字号、颜色一致 |
| 字距行距 | 与基准一致 |
| 玩家“我” | 坐标、朝向、动画帧一致 |
| 动态对象 | 坐标、层级、可见性和关键帧一致 |
| 遮挡关系 | 无错误遮挡、重叠、错层 |
| 转场状态 | 转场前后截图都需对比 |
| 失败反馈 | 失败提示、动画和状态截图需对比 |
| 闪白/震动/溶解 | 关键帧、强度和时长需对比 |

## 截图命名

推荐命名：

```text
harness/reports/visual/<level_id>/<baseline_id>__original.png
harness/reports/visual/<level_id>/<baseline_id>__replay.png
harness/reports/visual/<level_id>/<baseline_id>__diff.png
harness/reports/visual/<level_id>/<baseline_id>__report.json
```

## 通过条件

- `diff_pixel_count == 0`，或存在甲方批准的差异记录。
- 复刻截图必须来自自动演示或可重复脚本，不接受手动截图作为唯一证据。
- 差异报告必须记录 feature_id、baseline_id、截图路径、测试命令和生成时间。

## 失败条件

- 截图缺失。
- 复刻截图不是自动生成。
- 任一可见文字错字、漏字、多字。
- 任一对象位置、层级、大小或颜色不一致。
- 视觉差异未获批准。
- 视觉脚本返回非零退出码。
